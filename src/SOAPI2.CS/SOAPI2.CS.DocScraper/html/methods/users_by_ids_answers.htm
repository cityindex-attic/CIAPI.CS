Usage of /users/{ids}/answers

Discussion

Returns the answers the users in {ids} have posted.

{ids} can contain up to 100 separate ids, to find ids programatically look for user_id on user or shallow_user objects.

The sorts accepted by this method operate on the follow fields of the answer object:

  • activity – last_activity_date
  • creation – creation_date
  • votes – score
It is possible to create moderately complex queries using sort, min, max, fromdate, and todate.

This method returns a list of answers.
